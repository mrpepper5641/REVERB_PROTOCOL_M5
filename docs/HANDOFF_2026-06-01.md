## 体験フロー

```
[Introシーン]
  mainVisual.png を表示
  → M5接続待ち（AWAITING DEVICE...点滅）
  → 接続後「PRESS ANY BUTTON TO START」
  → BtnB or BtnC → フェードアウト → Mainシーンへ

[Mainシーン]
  Plane 0 (loginUI.png)
  → BtnB → ハックミニゲーム開始
  → ハック成功 → 自動で Plane 1 へ

  Plane 1 (logDisplay.png)
  → 自動表示
  → BtnC → Plane 2 へ

  Plane 2 (downloadWindow.png)
  → downloadDuration秒（デフォルト4秒）ダウンロード演出
  → DownloadCompleteUI表示（TRANSFER COMPLETE）
  → 終了
```

## 通信プロトコル

M5 → Unity:
```
D,{Pitch},{Roll},{AX},{AY},{AZ},{TX},{TY},{BtnA},{BtnB},{BtnC},{HackState},{HackProgress}\n
```
- TX/TY: タッチ座標（0.0〜1.0）、タッチなし = `-1`
- HackState: 0=IDLE, 1=INIT, 2=T1, 3=T2, 4=T3, 5=TL1, 6=TL2, 7=TL3, 8=SUCCESS, 9=FAIL

Unity → M5:
```
CMD,HACK_START\n
```

## シェーダープロパティ（C# から SetFloat する主要プロパティ）

| プロパティ名 | 意味 |
|---|---|
| `_ExtrudeAmount` | 法線方向への押し出し量（Pitch連動） |
| `_GhostOffset` | ゴーストのXYオフセット量（振り連動） |
| `_GlitchIntensity` | グリッチ強度 |
| `_TouchPoint` | タッチUV座標（Vector4） |
| `_TouchActive` | タッチ中かどうか（0 or 1） |
| `_MainTex` | テクスチャ（PNG） |
| `_TexBlend` | テクスチャブレンド（0=なし, 1=全面） |
| `_ShowHackButton` | ハックボタン表示（Plane0のみ） |
| `_HackButtonPulse` | ハックボタンパルスアニメ |

## M5サウンドシステム

| ch | 用途 |
|---|---|
| 0 | WAV再生（hack_start / success / fail） |
| 1 | UIトーン（ボタン・タッチ・ステップクリア） |
| 2 | Tiltホールドフィードバック |

- Shake音: **無効化済み**（コメントアウト）
- SDカード: `/sounds/` フォルダに以下を配置
  - `hack_start.wav` / `hack_step.wav` / `hack_success.wav` / `hack_fail.wav`
- ファイルなければ合成音にフォールバック

## 既知の課題・注意点

- Core2のBtnA/B/CはタッチYが240以上の領域として実装されており、`t.y < 240` でタッチ音とボタンを分離している
- `disturbanceThreshold = 0.5f`（これ以下だと普通に持っているだけでグリッチ発生）
- シェーダーは `Transparent` キュー・`ZWrite Off`・`Blend SrcAlpha OneMinusSrcAlpha`（テクスチャのアルファを使用）
- HLSLの予約語 `centroid` は使用禁止（`triCenter` に変更済み）

## 再開時の手順

1. Unity Hub から `REVERB_PROTOCOL_M5` を開く
2. M5 を USB-C で PC 接続
3. Arduino IDE は閉じておく
4. Introシーンから Play で動作確認
5. M5 を傾けて・タッチして・振って動作確認
