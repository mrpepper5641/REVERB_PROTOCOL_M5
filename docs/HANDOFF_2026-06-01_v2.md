## 体験フロー

```
[Introシーン]
  mainVisual.png を表示
  → M5接続待ち（AWAITING DEVICE...点滅）
  → 接続後「PRESS ANY BUTTON TO START」
  → BtnB or BtnC → フェードアウト → Mainシーンへ

[Mainシーン]
  Plane 0 (loginUI.png)
  → BtnB → ハックミニゲーム開始
  → ハック成功 → 自動で Plane 1 へ

  Plane 1 (logDisplay.png)
  → 自動表示
  → BtnC → Plane 2 へ

  Plane 2 (downloadWindow.png)
  → downloadDuration秒（デフォルト4秒）ダウンロード演出
  → DownloadCompleteUI表示（TRANSFER COMPLETE）
  → 終了
```

## 重要な実装変更点（前セッションで変更）

### M5Reader.cs — シングルトン化
```csharp
void Awake()
{
    if (Instance != null && Instance != this) { Destroy(gameObject); return; }
    Instance = this;
    DontDestroyOnLoad(gameObject);
}
```
→ Introシーンの M5Reader が Main に引き継がれる。Main シーンの M5Reader は自動破棄。

### M5Visual.shader — テクスチャUV反転
```glsl
float2 baseUV = float2(1.0 - i.uv.x, 1.0 - i.uv.y);
```
→ Unity Plane の UV が上下左右逆なので両軸を反転。

### M5Visual.shader — 2パス構成
- **Pass 0 (MAIN)**: テクスチャ表示 + 微細色収差（UV左右ズレ）+ タッチグロー
- **Pass 1 (GHOSTS)**: ジオメトリシェーダー義務のために存在、フラグメントは黒（無色）

### PostEffect.shader — スクリーンスペースグロー追加
- `_GhostOffset` に応じて輝度の高いピクセル周囲にR/B/Cyan を滲ませる
- M5VisualController → PostEffect.ghostOffset で連動

### HackMinigame.cs — UI調整済み
- %表示: バー下のステータス行に右揃え（パネル内・バー外）
- ハックボタン円リング: 無効化（ShowHackButtonID = 0 固定）

## 通信プロトコル

M5 → Unity:
```
D,{Pitch},{Roll},{AX},{AY},{AZ},{TX},{TY},{BtnA},{BtnB},{BtnC},{HackState},{HackProgress}\n
```
- TX/TY: タッチ座標（0.0〜1.0）、タッチなし = `-1`
- HackState: 0=IDLE, 1=INIT, 2=T1, 3=T2, 4=T3, 5=TL1, 6=TL2, 7=TL3, 8=SUCCESS, 9=FAIL

Unity → M5:
```
CMD,HACK_START\n
```

## シェーダープロパティ（C# から SetFloat する主要プロパティ）

| プロパティ名 | 意味 |
|---|---|
| `_ExtrudeAmount` | 法線方向への押し出し量（Pitch連動） |
| `_GhostOffset` | 色収差UV量（振り・グリッチ連動） |
| `_GlitchIntensity` | グリッチ強度 |
| `_TouchPoint` | タッチUV座標（Vector4） |
| `_TouchActive` | タッチ中かどうか（0 or 1） |
| `_MainTex` | テクスチャ（PNG） |
| `_TexBlend` | テクスチャブレンド（0=なし, 1=全面）← 全Plane 1 に設定済み |
| `_ShowHackButton` | ハックボタン表示（常に0、無効化済み） |

## 既知の課題・注意点

- Core2のBtnA/B/CはタッチYが240以上の領域として実装、`t.y < 240` でタッチ音とボタンを分離
- `disturbanceThreshold = 0.5f`
- `maxGhostOffset = 0.04f`（前セッションで下げた）
- HLSLの予約語 `centroid` は使用禁止（`triCenter` に変更済み）
- Introシーン Canvas は `ScreenSpaceCamera` モード（PostEffect適用のため）

## 再開時の手順

1. **Unity を先に起動**してプロジェクトを開く

3. **Claude Code を起動**
4. Unity Console でエラーがないか確認（前セッションのクラッシュ後のため）
5. M5 を USB-C で PC 接続、Arduino IDE は閉じておく
6. Introシーンから Play で動作確認
